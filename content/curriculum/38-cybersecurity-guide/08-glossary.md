# Cybersecurity Glossary

## Quick Reference Guide

A comprehensive glossary of essential cybersecurity terms, acronyms, and concepts for quick reference.

---

## A

**AAA (Authentication, Authorization, Accounting)**
Framework for controlling access: verifying identity (authentication), granting permissions (authorization), and tracking actions (accounting).

**Access Control**
Security measures that regulate who can view or use resources in a computing environment.

**Access Control List (ACL)**
List of permissions attached to an object specifying which users or processes can access it and what operations they can perform.

**Active Directory (AD)**
Microsoft's directory service for Windows domain networks, managing users, computers, and other resources.

**Advanced Persistent Threat (APT)**
Prolonged and targeted cyberattack in which an intruder gains access and remains undetected for an extended period, typically by nation-state actors.

**Adware**
Software that automatically displays or downloads advertising material when a user is online.

**AES (Advanced Encryption Standard)**
Symmetric encryption algorithm widely used worldwide, adopted by US government in 2001. Available in 128, 192, and 256-bit key sizes.

**Air Gap**
Security measure where a computer or network is physically isolated from unsecured networks, including the internet.

**Anomaly-Based Detection**
Intrusion detection method that identifies deviations from established baselines of normal behavior.

**Antivirus (AV)**
Software designed to detect, prevent, and remove malicious software (malware).

**API (Application Programming Interface)**
Set of protocols and tools for building software applications, defining how software components interact.

**APT (Advanced Persistent Threat)**
See Advanced Persistent Threat.

**Argon2**
Modern password hashing function, winner of Password Hashing Competition (2015), recommended for new applications.

**ARP (Address Resolution Protocol)**
Protocol for mapping IP addresses to physical MAC addresses on a local network.

**ARP Spoofing**
Attack technique where attacker sends falsified ARP messages to link their MAC address with IP of legitimate network device.

**Asset**
Anything of value to an organization that requires protection (data, systems, personnel, facilities).

**Asymmetric Encryption**
Encryption using key pairs (public and private keys) where data encrypted with one key can only be decrypted with the other. Also called public-key cryptography.

**Attack Surface**
Sum of all possible entry points where unauthorized user could try to enter or extract data from an environment.

**Attack Vector**
Path or means by which an attacker can gain access to a system to deliver a malicious payload or exploit.

**Audit Log**
Chronological record of system activities providing documentary evidence of operations, procedures, or events.

**Authentication**
Process of verifying the identity of a user, device, or system.

**Authorization**
Process of granting or denying specific permissions to access resources after authentication.

**Availability**
One of the CIA triad; ensuring information and resources are accessible to authorized users when needed.

---

## B

**Backdoor**
Method of bypassing normal authentication to gain access to a system, often installed by malware or intentionally by developers.

**Backup**
Copy of data stored separately from the original to enable recovery in case of data loss or corruption.

**Baseline**
Reference point representing normal operating conditions, used to detect anomalies and measure changes.

**Bcrypt**
Password hashing function based on Blowfish cipher, incorporating salt and configurable work factor.

**BIA (Business Impact Analysis)**
Process identifying critical business functions and potential impact of disruptions to determine recovery priorities.

**Biometric Authentication**
Verification of identity using biological characteristics (fingerprints, face, iris, voice).

**BIOS (Basic Input/Output System)**
Firmware that initializes hardware during boot process before loading the operating system.

**BitLocker**
Full disk encryption feature in Windows operating systems.

**Black Hat**
Hacker who violates computer security for malicious purposes or personal gain.

**Blacklist**
List of entities (IPs, domains, applications) that are explicitly denied access or blocked.

**Blockchain**
Distributed ledger technology using cryptographic hashing to create immutable records.

**Botnet**
Network of compromised computers controlled remotely by attacker, often used for DDoS attacks or spam distribution.

**BCP (Business Continuity Plan)**
Strategy ensuring essential business functions continue during and after disaster.

**Breach**
Incident where unauthorized individual gains access to confidential data, applications, networks, or devices.

**Brute Force Attack**
Attack method trying all possible combinations of passwords or encryption keys until correct one is found.

**Buffer Overflow**
Vulnerability where program writes more data to buffer than it can hold, potentially allowing code execution.

**Bug Bounty**
Program offering rewards to security researchers who find and report vulnerabilities.

---

## C

**CA (Certificate Authority)**
Trusted entity that issues digital certificates, verifying identity of certificate subjects.

**Caesar Cipher**
Simple substitution cipher where each letter is shifted by fixed number of positions in alphabet.

**CAPTCHA**
Challenge-response test to determine whether user is human (Completely Automated Public Turing test to tell Computers and Humans Apart).

**CCPA (California Consumer Privacy Act)**
California law giving consumers rights over their personal information collected by businesses.

**CERT (Computer Emergency Response Team)**
Group handling computer security incidents, providing alerts and guidance.

**CIA Triad**
Foundational information security model: Confidentiality, Integrity, Availability.

**Cipher**
Algorithm for performing encryption or decryption.

**Ciphertext**
Encrypted, unreadable form of data.

**CIS Controls**
Prioritized set of actions (20 controls) for cybersecurity defense by Center for Internet Security.

**CISO (Chief Information Security Officer)**
Executive responsible for organization's information and data security.

**Cleartext**
See Plaintext.

**Cloud Security**
Measures protecting data, applications, and infrastructure in cloud computing environments.

**COBIT**
Framework for governance and management of enterprise IT (Control Objectives for Information and Related Technology).

**Code Injection**
Attack inserting malicious code into vulnerable program, causing it to execute unintended commands.

**Code Signing**
Process of digitally signing executables and scripts to verify integrity and authenticate publisher.

**Cold Site**
Backup facility with minimal equipment, requiring days to weeks to become operational.

**Command and Control (C2/C&C)**
Server or infrastructure used by attacker to communicate with and control compromised systems.

**Compliance**
Adherence to laws, regulations, standards, and policies.

**Confidentiality**
One of the CIA triad; ensuring information is accessible only to authorized parties.

**Container Security**
Practices protecting containerized applications (Docker, Kubernetes) throughout their lifecycle.

**CORS (Cross-Origin Resource Sharing)**
Mechanism allowing web applications to make requests to different domain than the one serving the web page.

**Credential Stuffing**
Attack using lists of stolen usernames and passwords to gain unauthorized access, exploiting password reuse.

**CRL (Certificate Revocation List)**
List of digital certificates revoked before their expiration date.

**Cryptanalysis**
Study of analyzing and breaking cryptographic systems.

**Cryptography**
Science of securing information through mathematical techniques that transform readable data into unreadable data.

**Cryptojacking**
Unauthorized use of someone's computer to mine cryptocurrency.

**CSRF (Cross-Site Request Forgery)**
Attack forcing authenticated user to execute unwanted actions on web application.

**CSP (Content Security Policy)**
Security standard helping prevent cross-site scripting (XSS) and other code injection attacks.

**CVE (Common Vulnerabilities and Exposures)**
List of publicly disclosed cybersecurity vulnerabilities, each assigned unique identifier (CVE-ID).

**CVSS (Common Vulnerability Scoring System)**
Standardized method for rating severity of security vulnerabilities (0-10 scale).

---

## D

**DAC (Discretionary Access Control)**
Access control model where resource owner determines who can access resources.

**Dark Web**
Encrypted portion of internet not indexed by search engines, requiring special software (Tor) to access.

**Data Breach**
Incident where sensitive, protected, or confidential data is accessed or disclosed without authorization.

**Data Classification**
Process of categorizing data based on sensitivity and impact if compromised (e.g., Public, Internal, Confidential, Restricted).

**Data Exfiltration**
Unauthorized transfer of data from a computer or network.

**Data Loss Prevention (DLP)**
Strategy and tools preventing data loss, leakage, or misuse through monitoring, detection, and blocking.

**DDoS (Distributed Denial of Service)**
Attack overwhelming target with flood of internet traffic from multiple sources, making it unavailable.

**Defense in Depth**
Security strategy employing multiple layers of defense to protect assets.

**Denial of Service (DoS)**
Attack making system or network resource unavailable to intended users.

**DES (Data Encryption Standard)**
Obsolete symmetric encryption algorithm with 56-bit key, replaced by AES.

**DHCP (Dynamic Host Configuration Protocol)**
Network protocol automatically assigning IP addresses and network configuration to devices.

**Dictionary Attack**
Attack trying words from dictionary or list of common passwords to guess credentials.

**Diffie-Hellman**
Method for securely exchanging cryptographic keys over public channel.

**Digital Certificate**
Electronic document binding public key to identity of certificate owner.

**Digital Forensics**
Process of collecting, analyzing, and preserving electronic evidence for investigation.

**Digital Signature**
Cryptographic technique providing authentication, integrity, and non-repudiation for digital messages.

**DMZ (Demilitarized Zone)**
Network segment between internal network and external network (internet), hosting public-facing services.

**DNS (Domain Name System)**
System translating human-readable domain names to IP addresses.

**DNS Spoofing/Poisoning**
Attack corrupting DNS data to redirect traffic to malicious sites.

**DNSSEC (DNS Security Extensions)**
Suite of specifications securing DNS through cryptographic signatures.

**Domain**
Group of computers and devices on network administered as unit with common rules and procedures.

**DPI (Deep Packet Inspection)**
Network packet filtering examining data part of packet as it passes inspection point.

**DRP (Disaster Recovery Plan)**
Documented process for recovering IT systems and data after disaster.

**Dumpster Diving**
Practice of searching through trash to find confidential information.

---

## E

**ECC (Elliptic Curve Cryptography)**
Asymmetric encryption using elliptic curves, offering equivalent security to RSA with smaller key sizes.

**EDR (Endpoint Detection and Response)**
Security solution monitoring endpoints (workstations, servers) for suspicious activity and providing incident response capabilities.

**Egress Filtering**
Controlling and monitoring outbound network traffic from internal network to internet.

**Encryption**
Process of converting plaintext into ciphertext to protect confidentiality.

**Endpoint**
Any device connecting to network (computer, smartphone, tablet, IoT device).

**Endpoint Security**
Approach protecting endpoints from cyber threats through centralized management.

**Exploit**
Code or technique taking advantage of vulnerability to gain unauthorized access or cause unintended behavior.

**Exploit Kit**
Package of tools automating exploitation of vulnerabilities.

---

## F

**Fail-Safe**
Design ensuring system defaults to safe state in event of failure.

**Fail-Secure**
Design ensuring system defaults to secure state (denying access) in event of failure.

**False Negative**
Failure to detect actual threat or attack.

**False Positive**
Incorrectly identifying benign activity as malicious.

**Federated Identity**
Linking person's electronic identity across multiple separate identity management systems.

**FIDO (Fast Identity Online)**
Set of authentication standards reducing reliance on passwords (U2F, FIDO2, WebAuthn).

**FileVault**
Full disk encryption feature in macOS.

**Firewall**
Network security device monitoring and controlling incoming and outgoing traffic based on security rules.

**Firmware**
Permanent software programmed into hardware device's read-only memory.

**FISMA (Federal Information Security Management Act)**
US law requiring federal agencies to develop and implement information security programs.

**Forensics**
See Digital Forensics.

**FTP (File Transfer Protocol)**
Standard protocol for transferring files, but transmits data in cleartext (insecure; use SFTP/FTPS).

---

## G

**Gateway**
Device connecting two different networks, often with different protocols.

**GDPR (General Data Protection Regulation)**
EU regulation on data protection and privacy applying to processing of personal data.

**GRC (Governance, Risk, and Compliance)**
Strategy for managing organization's governance, risk management, and compliance with regulations.

**Gray Hat**
Hacker who may violate ethical standards or laws but doesn't have malicious intent of black hat.

**Group Policy**
Feature of Windows allowing centralized management and configuration of operating systems and applications.

---

## H

**Hacker**
Person who uses technical knowledge to overcome problem or security measure. Can be ethical (white hat) or malicious (black hat).

**Hacktivist**
Hacker motivated by political or social causes.

**Hash Function**
Algorithm converting input data of any size into fixed-size output (hash/digest), used for integrity verification.

**Hashing**
Process of using hash function to produce hash value from input data.

**Heartbleed**
Critical vulnerability (CVE-2014-0160) in OpenSSL library allowing attackers to read server memory.

**Heuristic Analysis**
Detection method using rules and algorithms to identify malicious behavior patterns.

**HIDS (Host-Based Intrusion Detection System)**
System monitoring individual host for suspicious activity.

**HIPAA (Health Insurance Portability and Accountability Act)**
US law establishing standards for protecting sensitive patient health information.

**HMAC (Hash-based Message Authentication Code)**
Cryptographic code combining hash function with secret key for authentication and integrity.

**Honeypot**
Decoy system designed to attract attackers, diverting them from legitimate targets and gathering intelligence.

**Hot Site**
Fully operational backup facility ready to take over operations immediately.

**HSTS (HTTP Strict Transport Security)**
Web security policy mechanism forcing browsers to use HTTPS connections only.

**HTTP (Hypertext Transfer Protocol)**
Application protocol for transmitting hypermedia documents (web pages), transmits in cleartext (insecure).

**HTTPS (HTTP Secure)**
Secure version of HTTP using TLS/SSL encryption to protect data in transit.

**Hybrid Cloud**
Computing environment combining public cloud, private cloud, and on-premises infrastructure.

---

## I

**IAM (Identity and Access Management)**
Framework for managing digital identities and controlling access to resources.

**IaaS (Infrastructure as a Service)**
Cloud computing model providing virtualized computing resources over internet.

**ICMP (Internet Control Message Protocol)**
Supporting protocol used for error messages and operational information (ping, traceroute).

**IDS (Intrusion Detection System)**
Device or software monitoring network for malicious activity and policy violations, generating alerts.

**IEC 62443**
Series of standards for industrial automation and control systems security.

**Incident**
Event that could lead to loss of, or disruption to, organization's operations or information.

**Incident Response**
Organized approach to addressing and managing aftermath of security breach or attack.

**Indicator of Compromise (IoC)**
Forensic evidence of potential intrusion or malicious activity on system or network.

**Information Security**
Practice of protecting information from unauthorized access, use, disclosure, disruption, modification, or destruction.

**Ingress Filtering**
Controlling and monitoring inbound network traffic from internet to internal network.

**Injection**
Class of attacks where untrusted data is sent to interpreter as part of command or query (SQL injection, command injection).

**Insider Threat**
Security risk originating from within organization (employees, contractors, partners).

**Integrity**
One of the CIA triad; ensuring information remains accurate and unaltered by unauthorized parties.

**Intrusion**
Unauthorized access to system, network, or data.

**IoC (Indicator of Compromise)**
See Indicator of Compromise.

**IoT (Internet of Things)**
Network of physical devices embedded with electronics, software, and sensors enabling connectivity and data exchange.

**IP (Internet Protocol)**
Protocol for routing and addressing packets of data across networks.

**IPS (Intrusion Prevention System)**
Network security device monitoring traffic and actively blocking detected threats.

**IPSec (Internet Protocol Security)**
Protocol suite for securing IP communications through authentication and encryption.

**ISO 27001**
International standard for information security management systems (ISMS).

**ISP (Internet Service Provider)**
Organization providing internet access services.

**IV (Initialization Vector)**
Random value ensuring unique ciphertext even when encrypting identical plaintext with same key.

---

## J

**Jailbreaking**
Process of removing software restrictions on iOS devices to gain root access.

**JavaScript**
Programming language commonly used in web development; can be exploited for XSS attacks.

**JWT (JSON Web Token)**
Compact, URL-safe means of representing claims between two parties, often used for authentication.

---

## K

**Kerberos**
Network authentication protocol using tickets to allow nodes to prove identity securely.

**Kernel**
Core component of operating system managing system resources and hardware communication.

**Key**
Secret value used with cipher for encryption and decryption.

**Key Exchange**
Method by which cryptographic keys are shared between parties.

**Key Management**
Activities involving handling of cryptographic keys throughout their lifecycle (generation, distribution, storage, rotation, destruction).

**Keylogger**
Malware or hardware device recording keystrokes to capture sensitive information like passwords.

**Kerckhoffs's Principle**
Cryptosystem should be secure even if everything except the key is public knowledge.

---

## L

**LAN (Local Area Network)**
Network connecting computers in limited area (home, office, building).

**Lateral Movement**
Techniques attackers use to progressively move through network, searching for target assets.

**LDAP (Lightweight Directory Access Protocol)**
Protocol for accessing and maintaining distributed directory information services.

**Least Privilege**
Security principle of granting users minimum access rights necessary to perform their job functions.

**Log**
Record of events occurring in operating system, application, or network.

**Log4Shell**
Critical vulnerability (CVE-2021-44228) in Apache Log4j logging library allowing remote code execution.

**Logic Bomb**
Malicious code activated when specific conditions are met (date, system event, etc.).

---

## M

**MAC (Mandatory Access Control)**
Access control system enforcing security policies based on centrally-assigned security labels.

**MAC Address (Media Access Control)**
Unique identifier assigned to network interface card.

**Malvertising**
Using online advertising to spread malware.

**Malware**
Malicious software designed to harm, exploit, or otherwise compromise system (viruses, worms, trojans, ransomware, spyware).

**MAN (Metropolitan Area Network)**
Network spanning geographic area larger than LAN but smaller than WAN (city-sized).

**Man-in-the-Middle (MitM)**
Attack where attacker intercepts communications between two parties without their knowledge.

**MD5 (Message Digest 5)**
Cryptographic hash function producing 128-bit hash value; considered broken, not for security use.

**Metadata**
Data providing information about other data (file creation date, author, location, etc.).

**MFA (Multi-Factor Authentication)**
Authentication method requiring two or more verification factors to gain access.

**Micorsegmentation**
Network security technique creating isolated zones to maintain separate access controls for different parts of network.

**MITM**
See Man-in-the-Middle.

**Mobile Device Management (MDM)**
Software managing and securing mobile devices (smartphones, tablets) used in enterprise.

---

## N

**NAC (Network Access Control)**
Approach to security allowing only compliant and authorized devices on network.

**NAT (Network Address Translation)**
Method remapping IP address space by modifying network address information in packet headers.

**Nation-State Actor**
Hacker or group sponsored by government to conduct cyberespionage or cyberwarfare.

**NGFW (Next-Generation Firewall)**
Advanced firewall combining traditional firewall with additional features (IPS, application awareness, deep packet inspection).

**NIDS (Network-Based Intrusion Detection System)**
System monitoring network traffic for suspicious activity.

**NIST (National Institute of Standards and Technology)**
US agency developing technology, metrics, and standards, including cybersecurity frameworks.

**NIST Cybersecurity Framework**
Voluntary framework for managing cybersecurity risk (Identify, Protect, Detect, Respond, Recover).

**Non-Repudiation**
Assurance that someone cannot deny validity of something (authorship of message, transaction).

**Nonce**
Arbitrary number used only once in cryptographic communication, preventing replay attacks.

---

## O

**OAuth**
Open standard for access delegation, commonly used for granting websites/applications access to user information without passwords.

**OCSP (Online Certificate Status Protocol)**
Protocol determining revocation status of digital certificate in real-time.

**OT (Operational Technology)**
Hardware and software controlling industrial operations (SCADA, ICS).

**OWASP (Open Web Application Security Project)**
Nonprofit organization improving software security, known for OWASP Top 10 web vulnerabilities.

**OWASP Top 10**
Standard awareness document listing most critical security risks to web applications.

---

## P

**PaaS (Platform as a Service)**
Cloud computing model providing platform allowing customers to develop, run, and manage applications.

**Packet**
Unit of data transmitted over network.

**Packet Sniffing**
Capturing and analyzing network packets to extract information.

**Parameterized Query**
Database query with placeholders for parameters, preventing SQL injection.

**Passphrase**
Sequence of words or text used for authentication, typically longer and more secure than password.

**Password**
Secret string of characters used for authentication.

**Password Manager**
Software helping generate, retrieve, and keep track of complex passwords securely.

**Password Spraying**
Attack trying commonly used passwords against many accounts to avoid account lockouts.

**Patch**
Software update addressing vulnerabilities or bugs.

**Patch Management**
Process of managing patches throughout development, testing, and deployment.

**Payload**
Malicious component of malware that performs harmful action.

**PCI DSS (Payment Card Industry Data Security Standard)**
Security standard for organizations handling credit card information.

**Penetration Testing (Pen Test)**
Authorized simulated attack on system to identify exploitable vulnerabilities.

**Perfect Forward Secrecy (PFS)**
Feature ensuring session keys are not compromised even if long-term private key is compromised.

**Perimeter Security**
Securing boundary between internal network and external networks (internet).

**Permissions**
Rules determining what actions users can perform on resources.

**Persistence**
Ability of malware to remain on system after reboot or attempts to remove it.

**Personally Identifiable Information (PII)**
Information that can identify specific individual (name, SSN, email, etc.).

**PGP (Pretty Good Privacy)**
Encryption program providing cryptographic privacy and authentication for data communication.

**Pharming**
Cyber attack redirecting website traffic to fake site to steal credentials.

**PHI (Protected Health Information)**
Health information that can be linked to specific individual, protected under HIPAA.

**Phishing**
Social engineering attack using fraudulent emails or websites to trick users into revealing sensitive information.

**PKI (Public Key Infrastructure)**
Framework for managing digital certificates and public-key encryption.

**Plaintext**
Readable, unencrypted data.

**Platform**
Operating system and/or hardware on which software runs.

**Policy**
Formal statement of rules governing security behavior and practices.

**Port**
Communication endpoint in operating system, identified by number (e.g., HTTP port 80, HTTPS port 443).

**Port Scanning**
Technique to discover open ports and services on networked device.

**Post-Quantum Cryptography**
Cryptographic algorithms secure against attacks by quantum computers.

**Pretexting**
Social engineering technique where attacker creates false scenario to obtain information.

**Privacy**
Right to control one's personal information and how it's used.

**Privilege**
Right to perform specific action on system or access resource.

**Privilege Escalation**
Exploiting vulnerability to gain elevated access beyond what's authorized.

**Protocol**
Set of rules governing data communication between devices.

**Proxy**
Intermediary server between client and destination server, can provide security, anonymity, and content filtering.

**Public Key**
Cryptographic key that can be shared publicly, used in asymmetric encryption.

**Private Key**
Secret cryptographic key that must be kept confidential, used in asymmetric encryption.

---

## Q

**Qualys**
Cloud-based security and compliance platform.

**Quarantine**
Isolating suspicious files or systems to prevent spread of malware.

**Quantum Computing**
Computing using quantum mechanics phenomena, posing threat to current public-key cryptography.

## R

**Rainbow Table**
Precomputed table of hash values for cracking password hashes.

**Ransomware**
Malware encrypting victim's data and demanding payment for decryption key.

**RAT (Remote Access Trojan)**
Malware providing attacker with remote control of infected system.

**RBAC (Role-Based Access Control)**
Access control model assigning permissions to roles rather than individual users.

**Red Team**
Group simulating attacks to test organization's security defenses.

**Remediation**
Process of fixing vulnerabilities or security weaknesses.

**Remote Code Execution (RCE)**
Vulnerability allowing attacker to execute code remotely on target system.

**Replay Attack**
Attack where valid data transmission is maliciously repeated or delayed.

**Residual Risk**
Risk remaining after controls and safeguards are applied.

**Reverse Proxy**
Proxy server retrieving resources from servers on behalf of client, providing security and load balancing.

**Risk**
Potential for loss or harm; calculated as likelihood × impact.

**Risk Assessment**
Process identifying and evaluating risks.

**Risk Management**
Process identifying, assessing, and controlling threats to organization.

**Rogue Access Point**
Unauthorized wireless access point on network.

**Rooting**
Process of gaining root/administrative access on Android device.

**Rootkit**
Malware providing privileged access while hiding its presence.

**RPO (Recovery Point Objective)**
Maximum acceptable amount of data loss measured in time.

**RSA**
Widely-used asymmetric encryption algorithm based on difficulty of factoring large prime numbers.

**RTO (Recovery Time Objective)**
Maximum acceptable time for system restoration after disruption.

## S

**SaaS (Software as a Service)**
Cloud computing model delivering applications over internet as service.

**Salt**
Random data added to passwords before hashing to prevent rainbow table attacks.

**SAML (Security Assertion Markup Language)**
XML-based standard for exchanging authentication and authorization data between parties.

**Sandbox**
Isolated environment for testing suspicious code or programs safely.

**SANS (SysAdmin, Audit, Network, and Security)**
Organization providing information security training and certifications.

**SCADA (Supervisory Control and Data Acquisition)**
Control system architecture for industrial processes.

**Scareware**
Malware using social engineering to cause shock or anxiety, manipulating users into buying unwanted software.

**Script Kiddie**
Unskilled individual using existing tools or scripts for hacking without understanding underlying concepts.

**Scrypt**
Password-based key derivation function designed to be memory-hard to resist hardware brute-force attacks.

**SDL (Security Development Lifecycle)**
Process integrating security into software development lifecycle.

**SecOps (Security Operations)**
Collaboration between security and IT operations teams.

**Secure Boot**
Security standard ensuring device boots using only trusted software.

**Secure Coding**
Practice of writing software resistant to security vulnerabilities.

**Secure Enclave**
Isolated processor providing additional security layer for sensitive operations (Apple devices).

**Security Awareness Training**
Programs educating users about security threats and best practices.

**Security Information and Event Management (SIEM)**
Solution providing real-time analysis of security alerts from applications and network hardware.

**Security Operations Center (SOC)**
Centralized unit monitoring, detecting, analyzing, and responding to cybersecurity incidents.

**Security Policy**
Documented rules and procedures governing security practices.

**Security Posture**
Overall cybersecurity strength of organization.

**Separation of Duties**
Security principle ensuring no single person has control over all aspects of critical transaction.

**Session**
Period during which user interacts with application after authentication.

**Session Hijacking**
Attack stealing or predicting session ID to impersonate user.

**Session Token**
Unique identifier assigned to user's session for maintaining authentication state.

**SFTP (SSH File Transfer Protocol)**
Secure file transfer protocol using SSH for encryption.

**SHA (Secure Hash Algorithm)**
Family of cryptographic hash functions (SHA-1, SHA-2, SHA-3); use SHA-256 or higher for security.

**Shadow IT**
IT systems and solutions used within organization without explicit approval.

**Shoulder Surfing**
Observing someone's screen or keyboard to obtain sensitive information.

**SIEM**
See Security Information and Event Management.

**Signature**
Pattern or characteristic used to identify specific malware or attack.

**Signature-Based Detection**
Detection method matching patterns against known threat signatures.

**SIM Swapping**
Attack transferring victim's phone number to attacker's SIM card to intercept calls and messages.

**Smishing**
Phishing attack conducted via SMS text messages.

**Sniffing**
Capturing and analyzing network packets.

**Snort**
Open-source network intrusion detection system (NIDS).

**SOC**
See Security Operations Center.

**SOC 2 (Service Organization Control 2)**
Audit report on service organization's controls related to security, availability, processing integrity, confidentiality, and privacy.

**Social Engineering**
Manipulation technique exploiting human psychology to gain access to systems or information.

**SOAR (Security Orchestration, Automation and Response)**
Technologies enabling organizations to collect security data and respond to security events with minimal human intervention.

**Software Bill of Materials (SBOM)**
Inventory of components used in software application.

**SOX (Sarbanes-Oxley Act)**
US law requiring public companies to maintain accurate financial records and internal controls.

**Spam**
Unsolicited messages sent in bulk, typically via email.

**Spear Phishing**
Targeted phishing attack directed at specific individual or organization.

**Spoofing**
Impersonating another device, user, or website to gain unauthorized access or deceive.

**Spyware**
Malware secretly monitoring user activity and collecting information.

**SQL (Structured Query Language)**
Language for managing and querying relational databases.

**SQL Injection (SQLi)**
Attack inserting malicious SQL code into query to manipulate database.

**SSH (Secure Shell)**
Cryptographic network protocol for secure remote access and command execution.

**SSL/TLS (Secure Sockets Layer / Transport Layer Security)**
Cryptographic protocols securing communications over networks; TLS is successor to SSL.

**SSRF (Server-Side Request Forgery)**
Vulnerability allowing attacker to make server send requests to unintended destinations.

**SSO (Single Sign-On)**
Authentication process allowing user to access multiple applications with one set of credentials.

**Steganography**
Hiding information within other non-secret data (images, audio, video).

**STRIDE**
Threat modeling framework: Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege.

**Stuxnet**
Sophisticated malware (2010) targeting industrial control systems, first known digital weapon.

**Subnet**
Logical subdivision of IP network.

**Supply Chain Attack**
Attack targeting less-secure elements in supply chain to compromise target organization.

**Symmetric Encryption**
Encryption using same key for encryption and decryption (AES, DES).

**SYN Flood**
DDoS attack exploiting TCP handshake by sending flood of SYN requests.

## T

**Tailgating**
Unauthorized person following authorized person into restricted area.

**TCP (Transmission Control Protocol)**
Core internet protocol providing reliable, ordered delivery of data.

**TCP/IP**
Suite of communication protocols used on internet and similar networks.

**Threat**
Potential danger that could exploit vulnerability to breach security.

**Threat Actor**
Entity responsible for security incident (individual, group, or nation-state).

**Threat Hunting**
Proactive searching for cyber threats lurking undetected in network.

**Threat Intelligence**
Information about threats and threat actors used to make informed security decisions.

**Threat Modeling**
Process identifying and evaluating potential threats during design phase.

**TLS**
See SSL/TLS.

**Token**
Physical or virtual object providing authentication (security token, session token).

**TOTP (Time-based One-Time Password)**
Algorithm generating one-time password from shared secret and current time.

**TPM (Trusted Platform Module)**
Hardware chip providing cryptographic functions and secure key storage.

**Trojan/Trojan Horse**
Malware disguised as legitimate software to trick users into installing it.

**Trust**
Assumption that entity will behave as expected.

**Two-Factor Authentication (2FA)**
Authentication requiring two different factors (something you know + something you have).

---

## U

**U2F (Universal 2nd Factor)**
Open authentication standard using hardware tokens for two-factor authentication.

**UEBA (User and Entity Behavior Analytics)**
Technology using machine learning to detect abnormal user and entity behavior.

**UDP (User Datagram Protocol)**
Connectionless transport protocol offering faster but unreliable data transmission compared to TCP.

**URL (Uniform Resource Locator)**
Web address specifying location of resource on internet.

**USB Drop Attack**
Social engineering attack leaving infected USB drive for someone to find and plug into computer.

---

## V

**Virus**
Malicious code that replicates by attaching itself to host files and spreads when files are executed.

**Vishing**
Phishing attack conducted via voice phone calls.

**VLAN (Virtual Local Area Network)**
Logical network division creating separate broadcast domains on same physical network.

**VPN (Virtual Private Network)**
Encrypted connection over internet providing secure remote access to network.

**Vulnerability**
Weakness in system that can be exploited by threat.

**Vulnerability Assessment**
Process identifying, quantifying, and prioritizing vulnerabilities.

**Vulnerability Management**
Practice identifying, evaluating, treating, and reporting on security vulnerabilities.

**Vulnerability Scanner**
Tool that probes systems for known vulnerabilities.

---

## W

**WAF (Web Application Firewall)**
Firewall protecting web applications by filtering and monitoring HTTP traffic.

**WAN (Wide Area Network)**
Network covering large geographic area, connecting multiple LANs.

**Warm Site**
Backup facility with equipment and connectivity but may require data updates to become operational.

**Watering Hole Attack**
Attack compromising websites frequently visited by target group.

**Web Shell**
Script uploaded to web server providing remote access and control.

**WebAuthn**
Web standard for authenticating users using public key cryptography, part of FIDO2.

**Whaling**
Spear phishing attack targeting high-profile individuals (executives).

**White Hat**
Ethical hacker who uses skills to improve security with permission.

**Whitelist**
List of entities explicitly permitted access or allowed to execute.

**Wi-Fi Protected Access (WPA/WPA2/WPA3)**
Security protocols for wireless networks; use WPA3 or WPA2 minimum.

**WireGuard**
Modern VPN protocol emphasizing simplicity and performance.

**Worm**
Self-replicating malware spreading independently across networks without user interaction.

---

## X

**XDR (Extended Detection and Response)**
Security approach providing unified visibility and response across multiple security layers.

**XACML (eXtensible Access Control Markup Language)**
XML-based standard for expressing security policies.

**XML (eXtensible Markup Language)**
Markup language defining rules for encoding documents.

**XSS (Cross-Site Scripting)**
Vulnerability allowing attackers to inject malicious scripts into web pages viewed by other users.

**XXE (XML External Entity)**
Attack exploiting XML parsers to access local files or internal networks.

---

## Y

**YARA**
Tool for identifying and classifying malware based on textual or binary patterns.

**YubiKey**
Hardware authentication device supporting multiple authentication protocols.

---

## Z

**Zero-Day**
Vulnerability unknown to vendor and public, with no available patch; or exploit for such vulnerability.

**Zero Trust**
Security model requiring verification for every access request regardless of location (never trust, always verify).

**Zombie**
Compromised computer controlled remotely as part of botnet.

This glossary provides quick reference for essential cybersecurity terminology. Understanding these terms is fundamental for effective communication in cybersecurity contexts, whether you're a security professional, developer, IT administrator, or someone interested in protecting digital assets.

The field continuously evolves with new threats, technologies, and terminology emerging regularly. Consider this glossary a living document—new terms will arise as cybersecurity advances. For the most current information on specific vulnerabilities, consult resources like CVE databases, vendor security advisories, and reputable cybersecurity news sources.
