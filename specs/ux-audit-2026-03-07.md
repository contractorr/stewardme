# UX Audit Report — 2026-03-07

## Scope

- Compared `specs/functional/*.md` with the current dashboard routes in `web/` and supporting FastAPI routes in `src/web/routes/`.
- Reviewed the highest-priority workflows requested for this iteration:
  - onboarding
  - analytics/admin
  - goal tracking
  - projects/opportunities
  - settings/account

## Audit Summary

### Resolved in this iteration

1. **Projects/opportunities had backend support but no dashboard workspace**
   - Before: `/api/projects/issues` and `/api/projects/ideas` existed, but the dashboard had no route or navigation entry.
   - Now: a dedicated `/projects` workspace ships in the sidebar with tactical issue matches and strategic idea generation.

2. **Settings save affordance lived too far from the edits that trigger it**
   - Before: provider and key edits were made high on the page, but save lived near the bottom.
   - Now: unsaved provider/key changes trigger a sticky save bar and top-of-page section jump links.

3. **Settings functional spec under-described current product scope**
   - Before: the spec focused on keys and deletion, but omitted RSS feeds, watchlist management, and structured profile editing.
   - Now: the spec reflects the shipped settings surface.

4. **Goal-tracking functional spec lagged behind the shipped goals workspace**
   - Before: the spec omitted weekly-plan capacity and linked recommendation action items.
   - Now: the spec documents both behaviors.

5. **Analytics/admin spec under-described current admin page behavior**
   - Before: the spec did not call out fixed day-window controls or smaller-screen table handling.
   - Now: the spec reflects the shipped filtering and responsive behavior.

## Prioritized UX Issues

### P0

- **Missing projects workspace** — high user impact because a documented capability had no visible dashboard path.

### P1

- **Long-scroll settings save friction** — high frequency, moderate severity; easy to create unsaved-state anxiety.
- **Analytics tables on small screens** — moderate severity for admins; data remained available but harder to scan.

### P2

- **Goals spec mismatch** — low immediate product risk, high documentation drift risk.
- **Settings spec mismatch** — moderate documentation risk; likely to mislead future implementation work.

## Screen Redesign Notes

### Onboarding

- Current flow remains strong: intro, name, key/lite choice, interview, feed setup, guide, completion.
- No code changes landed in this iteration.
- Next likely improvement: compress the welcome/key step on mobile and make progress more explicit during the interview.

### Analytics / Admin

- Use a shared workspace header so the page reads like the rest of the dashboard.
- Keep day-window controls and refresh action in the header area.
- Wrap tables in horizontal overflow rather than allowing values to compress into unreadability.

### Goal Tracking

- Current page already prioritizes focus goals, stale goals, and archived goals well.
- Documentation now reflects the shipped weekly-plan and action-item behaviors.
- Next likely improvement: reduce vertical expansion cost for dense goal cards on mobile.

### Projects / Opportunities

- Split the workspace into two tracks:
  - issue matches for fast execution
  - project ideas for broader exploration
- Keep issue filters lightweight and visible.
- Preserve useful empty states when radar data is thin.

### Settings / Account

- Add section jump chips to reduce orientation cost across a long page.
- Use a sticky save bar for pending provider/key changes.
- Keep destructive actions isolated in their own section.

## Simulated Workflow Notes

- **Onboarding:** still documented as the main gate for users without profile context.
- **Analytics/admin:** improved for small-screen readability through scroll-safe tables and stronger page framing.
- **Goal tracking:** docs now align with weekly-plan and linked action-item reality.
- **Projects/opportunities:** workflow moved from API-only to a visible dashboard path.
- **Settings/account:** click depth to save edits is reduced because save/discard actions stay persistent once changes exist.

## Remaining Improvements

- Add richer cross-linking from project ideas into goals creation.
- Add automated browser regression coverage for the new `/projects` route.
- Revisit mobile density on the goals page.
- Consider a shared empty-state component if more dashboard pages adopt the same pattern.
