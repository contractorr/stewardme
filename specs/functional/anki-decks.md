# Anki Decks (Flashcard Import / Export)

## Problem

Users who study with Anki have years of flashcards locked in `.apkg` files, and users of the
Learn workspace have review material locked in the app. There is no bridge: imported knowledge
cannot join the app's review queue, and app-generated review questions cannot travel to Anki.
The app should embed Anki-style flashcard functionality so users can import decks, review cards
on the app's existing spaced-repetition schedule, author new cards, and export decks (including
app-generated review questions) back to Anki.

## Users

- Existing Anki users who want their decks reviewable inside the Learn workspace.
- Learn users who want to take app-generated review questions with them into Anki.
- Users who want simple front/back flashcards without authoring a full guide.

## Desired Behavior

### Import

1. User opens the Decks area under Learn and chooses `Import deck`.
2. User uploads an `.apkg` file exported from Anki.
3. The system parses the deck, creates a named deck, and converts each Anki card into a
   flashcard with front/back text. Prior scheduling state (ease, interval, repetition count,
   due date) is preserved so the user does not restart their review history.
4. The user sees the imported deck with its card count and how many cards are due.
5. Media files (images, audio) are skipped in this version; the user is told how many
   media references were dropped.

### Review

1. Cards from all decks join a flashcard review queue scheduled by the same SM-2
   spaced-repetition engine used for chapter reviews.
2. The Learn page's review surface includes due flashcards in its due count.
3. A flashcard review session shows the front, the user reveals the back, then self-grades
   with Anki-style buttons (`Again`, `Hard`, `Good`, `Easy`).
4. Grading reschedules the card and advances to the next due card.

### Authoring

1. User can create an empty deck with a title and description.
2. User can add, edit, and delete individual front/back cards in a deck they own.

### Export

1. User can export any deck as an `.apkg` file that imports cleanly into Anki.
2. User can export their chapter review questions (quiz items generated by the app) as an
   `.apkg` deck, so app learning can continue in Anki.
3. Exported cards are exported as new cards (Anki's convention for shared decks);
   scheduling state is not exported.

## Acceptance Criteria

- [ ] Uploading a valid `.apkg` (Anki 2.0/2.1 SQLite format) creates a deck whose card count
      matches the source deck's cards.
- [ ] Imported cards preserve ease factor, interval, and repetition count; previously due
      cards are due on import.
- [ ] Uploading a non-apkg file or a corrupt archive shows a clear error and creates nothing.
- [ ] Uploading a newer Anki export format that is not supported (zstd-compressed
      `collection.anki21b` only) shows a message telling the user to re-export with
      "support older Anki versions" enabled.
- [ ] Due flashcards appear in a review queue and grading with `Again/Hard/Good/Easy`
      reschedules the card (SM-2).
- [ ] `Again` on a card makes it due again within the same day; `Easy` pushes it further out
      than `Good`.
- [ ] A deck can be exported to `.apkg` and re-imported, yielding the same fronts/backs.
- [ ] Chapter review questions can be exported as an `.apkg` deck with question on the front
      and expected answer on the back.
- [ ] Decks and cards are per-user: another user cannot list, review, or export them.
- [ ] Deleting a deck removes its cards from the review queue.

## Edge Cases

| Scenario | Expected Behavior |
| --- | --- |
| `.apkg` with multiple sub-decks | All cards import into one app deck named after the file/top deck; sub-deck names are kept on cards as tags where available |
| Cards with HTML formatting (bold, lists, cloze) | Rendered to readable plain text; cloze deletions shown with the blank marker on the front and the full text on the back |
| Card with empty front after HTML stripping | Card is skipped and counted in the import summary |
| Very large deck (thousands of cards) | Import completes and reports the count; review queue stays capped per session |
| Deck with media references | Cards import without media; import summary reports skipped media count |
| Export of an empty deck | Valid `.apkg` with zero cards |
| User has no decks | Decks area shows a quiet empty state with Import and New deck actions |
| Duplicate import of the same file | A second deck is created (no silent merge); the user can delete either |

## Out of Scope

- Media (images/audio) import or export.
- Anki template rendering beyond basic field substitution and cloze flattening.
- Two-way sync with AnkiWeb or the Anki desktop client.
- Exporting SM-2 scheduling state back to Anki.
- LLM generation of flashcards from notes or chapters (future work; see `note-polish`).

## Validation Notes

- Smallest meaningful validation slice: round-trip test — build an `.apkg` in-memory,
  import it, review a card, export it, re-import it.
- Contract impact: new `/api/curriculum/decks/*` and `/api/curriculum/review/flashcards/*`
  endpoints; new frontend types.
- Follow-up spec work: flashcard generation from polished notes and chapters.
